### 2. What Is Actually Built and Working
*The following features are wired into the active execution path and compile successfully.*

- **Interactive REPL Loop:** Reads stdin, dispatches slash commands, and sends API requests (`src/main.rs`, `src/runtime.rs`).
- **NIM Provider Integration:** Formats API requests for NVIDIA NIM and parses SSE streams (`src/main.rs`, `src/provider/`, `src/sse.rs`).
- **JSONL Session Persistence:** Serializes and loads session history from `.claw/sessions/` (`src/session.rs`).
- **Tool Dispatch Engine:** Executes a subset of system tools and handles responses (`src/tools/dispatch.rs`).
- **Filesystem & Shell Tools:** `shell`, `file_read`, `file_write`, `file_edit`, `glob_search`, and `grep_search` are fully implemented.
- **Python Companion Scaffold:** Exposes CLI subcommands for routing, session loading, parity auditing, and querying (`python_companion/main.py`).

### 5. Data Contracts (Frozen)
*Extracted from `src/data_contracts.rs`*

- **Content Blocks (`ContentBlock`):** JSON tagged (`type`) with `text`, `tool_use`, `tool_result`, `thinking`, `redacted_thinking`.
- **Session Records (`SessionMessageRecord`):** Contains `role` (System, User, Assistant, Tool), `content` (Array of blocks), `usage` (Nullable), `timestamp`, `tool_call_id` (Nullable).
- **Session Metadata (`SessionMetadata`):** Always contains `session_id`, `created_at`, `model`, `permission_mode`, `heartbeat`, `liveness` (boolean), `compaction_history` (Array).
- **Worker State (`WorkerState`):** `worker_id`, `session_id`, `model`, `permission_mode`.

### 6. Known Debt & Inaccuracies
*(Verified directly from source tree stubs and missing implementations)*

- **Anthropic Provider Stubbed:** `src/main.rs` hardcodes an error message if `anthropic` is selected ("not yet wired in Stage 3"). (Feature Gap)
- **Missing Providers:** OpenAI, xAI, and DashScope (claimed in README) are entirely absent from `src/main.rs` and `src/provider/`. (Doc Error / Feature Gap)
- **Stubbed Tools:** `src/tools/agent_launch.rs`, `notebook_edit.rs`, `todo_write.rs`, `skill_invoke.rs`, `web_search.rs`, and `web_fetch.rs` are hardcoded to return `unimplemented!()` or "NotYetImplemented". (Feature Gap)
- **Stubbed Python Modes:** `python_companion/main.py` explicitly returns `{"detail": "not implemented"}` for `remotemode`, `sshmode`, `teleportmode`, `directconnectmode`, and `deeplinkmode`. (Feature Gap)
- **Hook Lifecycle Missing:** While `src/hook.rs` is parsed by configuration, no hooks are actually invoked during tool execution in `src/tools/dispatch.rs` or `src/runtime.rs`. Hooks are effectively scaffolded dead-code. (Feature Gap)

### 8. Build & Run Reference

**Build the Rust Core (Working):**
```bash
cargo build --release
```

**Run the Rust REPL (Working):**
```bash
# Ensure NAKAMA_PROVIDER=nim and NVIDIA_API_KEY are set in .env
cargo run
```

**Run the Python Companion (Working):**
