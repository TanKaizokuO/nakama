# Nakama

## Overview
**Vision/Goal:** Nakama is a dual-architecture AI coding assistant and agent infrastructure featuring a high-performance primary engine in Rust and a Python companion workspace that handles state mirroring, prompt routing, and parity auditing.

**Current Status:** Active Development

## Tech Stack
**Language/Runtime:** Rust (Edition 2024), Python (3.10+)

**Frameworks/Libraries:** 
- Rust: `tokio` (async runtime), `reqwest` (HTTP), `serde` / `serde_json` (serialization), `clap` (CLI), `rustyline` (REPL)
- Python: Standard library (argparse, dataclasses, json)

**Key Dependencies:** `eventsource-stream` (SSE handling), `uuid`, `chrono` (timestamps/identifiers), `thiserror` (error handling).

## Core Logic & Data Flow
1. **Conversation Loop & Runtime (Rust)**: The core user interaction is managed by `ConversationRuntime` (`src/runtime.rs`). When a user inputs a prompt, the runtime parses for slash commands, registers the message, and triggers the compaction engine (`CompactionEngine`) to ensure token limits aren't breached. It then simulates sending the request to a provider, streaming Server-Sent Events (SSE), and evaluating tool usage.
2. **State Persistence & Worker Management**: As the turn executes, `Session` saves the conversation state to disk in a strict `.jsonl` structure. `WorkerStateManager` writes a session marker representing the active parameters (model, permission mode, session id) to track execution contexts.
3. **Companion Routing & Parity Auditing (Python)**: The Python workspace consumes artifacts generated by Rust. `RoutingEngine` processes unstructured text to score and route commands. `ParityAuditor` validates that tools and command entries remain perfectly in sync with the primary Rust binary to prevent system drift.


## Development Conventions
- **Strict Data Contracts:** All internal state, token usage, and pricing details use heavily typed Rust structs (`data_contracts.rs`).
- **Permission Boundaries:** Tool execution and filesystem access are tightly guarded by explicitly defined permission modes (`ReadOnly`, `WriteApprove`).
- **JSONL Persistence:** Message transcripts and session metadata are strictly maintained in `.jsonl` format, allowing the Python companion to seamlessly load and audit them.
- **Deduplicated Bootstrapping:** The Rust engine uses a rigid 12-stage pipeline to initialize the environment safely, checking for sandboxes (Docker, WSL) before allowing tool execution.

## Known Issues / Debt
- **Mocked Provider Integration:** Current implementations in `src/runtime.rs` (like provider API responses and tool stop-reasons) contain mocked values for simulation that require completion against live endpoints.
- **Dual-Architecture Drift Risk:** Because logic is mirrored in the Python companion for parity auditing, changes to Rust CLI hooks, tools, or data contracts must be continuously tested via `python main.py parityaudit` to prevent the architectures from desyncing.
- **Hardcoded Context Limits:** Token boundaries during compaction fallback to hard limits (e.g. 100,000) rather than dynamically reading limits based on the active model.
